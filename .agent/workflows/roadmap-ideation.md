---
description: Transform a vague idea or a "problem statement" into a prioritized feature set.
---

### 1. Opportunity Discovery (Product Strategist)
**Action**: Analyze the current state of the project (Holocron or Macro Tracker).
**Focus**: 
- What is the biggest pain point for current users?
- What "low-hanging fruit" exists that would provide 10x value for 1x effort?
- Generate 3 distinct feature "themes" (e.g., "Automation," "Community," "Privacy").

### 2. Technical Feasibility (Senior Software Engineer)
**Action**: High-level "T-shirt sizing" (Small, Medium, Large) for each theme.
**Focus**:
- Which ideas require a total database schema rewrite?
- Which ideas can be built as a modular "bolt-on"?
- Identify any "Technical Debt" that must be paid before these features can ship.

### 3. Impact vs. Effort Matrix (Product Strategist)
**Action**: Plot the ideas on a 2x2 matrix.
**Output**: A prioritized "Backlog" of features categorized as:
- **Quick Wins**: High Impact, Low Effort.
- **Big Bets**: High Impact, High Effort.
- **Fill-ins**: Low Impact, Low Effort.
- **Thankless Tasks**: Low Impact, High Effort (Avoid these).
