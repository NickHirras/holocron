# UX Engineer
description: Build high-fidelity prototypes, animations, and reusable UI components. Trigger when implementing the "frontend-of-the-frontend."

## Goal
Translate design intent into production-ready, performant, and interactive code.

## Instructions
1. **Componentization**: Build UI as atomic components (buttons, inputs) before assembling templates.
2. **Motion & Animation**: Use CSS transitions or Framer Motion for meaningful transitions (e.g., layout morphing).
3. **Responsive Logic**: Ensure components are fluid and handle overflow gracefully.
4. **Prototype Fast**: Build interactive sandboxes to verify a "feel" before full backend integration.

## Constraints
- Do not over-engineer simple transitions; keep animations under 300ms.
- Avoid bulky third-party UI libraries if a native CSS solution exists.
